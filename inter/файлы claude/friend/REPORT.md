## 2. Синтаксис реализованного языка (расширенная БНФ)

```
〈программа〉     → program { 〈описания〉 〈операторы〉 }

〈описания〉      → { 〈описание〉 ; }
〈описание〉      → 〈тип〉 〈переменная〉 { , 〈переменная〉 }
〈тип〉           → int | string | real

〈переменная〉    → 〈идентификатор〉
                 | 〈идентификатор〉 = 〈константа〉

〈константа〉     → 〈целочисленная〉 | 〈строковая〉 | 〈вещественная〉
〈целочисленная〉 → [〈знак〉] 〈цифра〉 { 〈цифра〉 }
〈вещественная〉  → [〈знак〉] 〈цифра〉 { 〈цифра〉 } . 〈цифра〉 { 〈цифра〉 }
〈строковая〉     → " { 〈литера〉 } "
〈знак〉          → + | -

〈операторы〉     → { 〈оператор〉 }
〈оператор〉      → if ( 〈выражение〉 ) 〈оператор〉 [ else 〈оператор〉 ]
                 | for 〈идентификатор〉 = 〈выражение〉
                     step 〈выражение〉 until 〈выражение〉 do 〈оператор〉
                 | while ( 〈выражение〉 ) 〈оператор〉
                 | 〈помеченный оператор〉
                 | goto 〈идентификатор〉 ;
                 | read ( 〈идентификатор〉 ) ;
                 | write ( 〈выражение〉 { , 〈выражение〉 } ) ;
                 | { 〈операторы〉 }
                 | 〈оператор-выражение〉

〈помеченный оператор〉 → 〈идентификатор〉 : 〈оператор〉
〈оператор-выражение〉  → 〈выражение〉 ;

〈выражение〉     → 〈присваивание〉
〈присваивание〉  → 〈идентификатор〉 = 〈присваивание〉 | 〈или〉
〈или〉           → 〈и〉 { or 〈и〉 }
〈и〉             → 〈отношение〉 { and 〈отношение〉 }
〈отношение〉     → 〈сложение〉 { ( < | > | <= | >= | == | != ) 〈сложение〉 }
〈сложение〉      → 〈умножение〉 { ( + | - ) 〈умножение〉 }
〈умножение〉     → 〈унарное〉 { ( * | / | % ) 〈унарное〉 }
〈унарное〉       → not 〈унарное〉 | - 〈унарное〉 | 〈первичное〉
〈первичное〉     → 〈целочисленная〉 | 〈вещественная〉 | 〈строковая〉
                 | 〈идентификатор〉 | ( 〈выражение〉 )
```

**Синтаксис идентификатора:**
```
〈идентификатор〉 → 〈буква〉 { 〈буква〉 | 〈цифра〉 | _ }
〈буква〉         → [a-z] | [A-Z]
```

## 3. Неопределённые аспекты и принятые решения

| Аспект | Принятое решение |
|--------|-----------------|
| Максимальная длина идентификатора | Не ограничена (`std::string`) |
| Формат вывода `real` | Убираем trailing-нули: `3.14000` → `3.14` |
| Формат ввода `real` | Стандартный `cin >> double` |
| `int = real` при присваивании | Усечение: `3.9 → 3` |
| `real = int` при присваивании | Неявное расширение: `3 → 3.0` |
| `write` — разделитель | Пробел между значениями |
| Унарный минус к строке | Ошибка времени выполнения |
| Параметр цикла `for` | Обязан быть типа `int` (требование ТЗ п.5) |
| Хранение E2, E3 в for | Скрытые переменные `__step_N`, `__limit_N` |
| Значение параметра after for | Не определено (ТЗ: "считается неопределённым") |
| `else` | Привязывается к ближайшему `if` |
| Метки | Объявляются использованием: `имя: оператор` |
| Метки — пространство имён | Отдельное (метки и переменные не пересекаются) |

## 4. Конечный автомат лексического анализатора

```
НАЧАЛО
  ├── пробел/\n/\t     → НАЧАЛО (пропустить)
  ├── '/' + '*'        → КОММЕНТАРИЙ
  │     └── '*' + '/'  → НАЧАЛО
  ├── '"'              → СТРОКА
  │     ├── символ     → СТРОКА (накапливаем)
  │     └── '"'        → КОНЕЦ: STR_LIT
  ├── цифра [0-9]      → ЦЕЛОЕ
  │     ├── цифра      → ЦЕЛОЕ (накапливаем)
  │     └── '.' + цифра→ ВЕЩЕСТВЕННОЕ
  │           ├── цифра→ ВЕЩЕСТВЕННОЕ
  │           └── иное → КОНЕЦ: REAL_LIT
  ├── буква/_          → ИДЕНТИФИКАТОР
  │     ├── буква/цифра/_ → ИДЕНТИФИКАТОР
  │     └── иное       → КОНЕЦ: сверить с ключ. словами (for/step/until/goto/...)
  ├── '<' + '='        → КОНЕЦ: LE
  ├── '>' + '='        → КОНЕЦ: GE
  ├── '=' + '='        → КОНЕЦ: EQ
  ├── '!' + '='        → КОНЕЦ: NEQ
  ├── ':'              → КОНЕЦ: COLON  ← НОВОЕ (разделитель метки)
  └── одиночный        → КОНЕЦ: соответствующий токен
```

## 5. Обоснование применимости метода рекурсивного спуска

Грамматика **не содержит леворекурсии** (ни прямой, ни косвенной).

**FIRST-множества операторов** (нет пересечений → LL(1)):
```
FIRST(if-stmt)         = { if }
FIRST(for-stmt)        = { for }
FIRST(while-stmt)      = { while }
FIRST(goto-stmt)       = { goto }
FIRST(помеченный-stmt) = { ID } при условии look-ahead(':')
FIRST(read)            = { read }
FIRST(write)           = { write }
FIRST(блок)            = { { }
FIRST(expr-stmt)       = { ID без ':', INT, REAL, STR, (, -, not }
```

**Особый случай — разрешение неоднозначности ID:**
Когда текущий токен `ID`, нужен look-ahead на 1 токен:
- Следующий токен `':'`  → это помеченный оператор
- Следующий токен `'='`  → это присваивание в выражении
- Иначе                  → это выражение-оператор

Метод рекурсивного спуска поддерживает look-ahead через `peekAhead()`.

**Левоассоциативные операции** (`+`, `-`, `*`, `/`, сравнения) реализованы через `while`-циклы (не рекурсию), что устраняет левую рекурсию.

## 6. Генерация ПОЛИЗ для ключевых конструкций

### for I = E1 step E2 until E3 do S

```
Требование ТЗ: E1, E2, E3 вычисляются ОДИН РАЗ до входа в цикл.

ПОЛИЗ:
  [E1]                     ← вычисляем начальное значение
  STORE I                  ← I = E1
  POP
  [E2]                     ← вычисляем шаг
  STORE __step_N
  POP
  [E3]                     ← вычисляем предел
  STORE __limit_N
  POP

check:
  LOAD I
  LOAD __limit_N
  GT                       ← I > limit?
  JNZ → end               ← если да, выходим
  [S]                      ← тело цикла
  LOAD I
  LOAD __step_N
  ADD
  STORE I                  ← I = I + step
  POP
  JMP → check
end:
```

### goto (forward и backward)

```
Backward goto (метка уже известна):
  goto myLabel;  →  JMP  [адрес myLabel]   ← адрес уже в таблице

Forward goto (метка ещё не встречена):
  goto future;   →  JMP  0                 ← адрес = 0 (заглушка)
                    (индекс запомнен в fwdGotos["future"])

При встрече метки:
  future: stmt   →  регистрируем future → here()
                    патчим все fwdGotos["future"] → here()
```

### Обычные вычисления AND / OR

```
Обычные (вариант VI.2) — оба операнда ВСЕГДА вычисляются:

  a and b:           a or b:
    [a]                [a]
    [b]                [b]
    AND                OR

Инструкция AND: снять R и L; push (L≠0 && R≠0) ? 1 : 0
Инструкция OR:  снять R и L; push (L≠0 || R≠0) ? 1 : 0

Отличие от ленивых: нет JZ/JMP — правый операнд
вычисляется всегда, даже если результат уже ясен из левого.
```

## 7. Объектная модель

```
┌─────────────┐   tokenize()   ┌──────────────────┐
│   Lexer     │ ─────────────→ │  vector<Token>   │
│             │                └──────────────────┘
│ src: string │                         │
│ pos: size_t │                         ▼
│ ln:  int    │           ┌─────────────────────────┐
└─────────────┘           │         Parser          │
                          │                         │
┌─────────────┐           │ toks:     vector<Token> │
│  SymTable   │◄──────────│ sym:      SymTable&     │
│             │           │ code:     vector<Poliz..│
│ vars: map   │           │ labels:   map<str,int>  │
│  (перем.)   │           │ fwdGotos: map<str,vec>  │
└─────────────┘           │ forCounter: int         │
      ▲                   └─────────────────────────┘
      │                                │
      │                    ┌──────────────────────┐
      │                    │  vector<PolizOp>     │
      │                    │  (программа в ПОЛИЗ) │
      │                    └──────────────────────┘
      │                                │
      │                   ┌────────────────────────┐
      └───────────────────│     Interpreter        │
                          │                        │
                          │ code: vector<PolizOp>& │
                          │ sym:  SymTable&         │
                          │ st:   stack<Val>        │
                          └────────────────────────┘

Val = variant<long long, double, string>  (C++17)
```

**Используемые возможности C++:**
- `std::variant<long long, double, std::string>` — типобезопасное хранение значений
- `std::unordered_map` — таблица символов, таблица меток, forward-goto
- `std::stack<Val>` — рабочий стек интерпретатора
- `std::runtime_error` — все ошибки (лексика, синтаксис, семантика, runtime)
- Конструкторы и инкапсуляция в каждом классе

## 8. Комплект А-тестов (правильные программы)

### A1 — базовый тест из ТЗ (общая часть)
```
program {
    int a = 51, b = 6, c;
    string x = "abc", y, z = "abcd";
    c = (a + b) * 2;
    if (c >= 100 or x == z) {
        read (y);
        write (y);
        write (x + y + z, c);
    }
    else
        c = a = 21;
    while (c > 100) {
        c = c - 5;
        write (c);
        x = x + "step";
    }
    write (x);
}
```
Вход: `1234`
Ожидаемый: `1234 abc1234abcd 114 109 104 99 abcstepstepstep`
✅ Получено: `1234 abc1234abcd 114 109 104 99 abcstepstepstep`

}

## 10. Сборка и запуск

```bash
# Компиляция
g++ -std=c++17 -Wall -o interp main.cpp

# Запуск из файла
./interp program.ml

# С вводом из файла (если программа использует read)
./interp program.ml < input.txt

# Или интерактивно — вводить с клавиатуры
./interp program.ml
```
